Distributed Graph Store

Alexis Saiontz & Stelios Rousoglou
CPSC 426: Building Distributed Systems

Summary

Over the course of a semester, we collaborated on four iterations of a graph store that handles graph requests.

First iteration: Simple HTTP API for an in-memory graph

This iteration stores undirected graphs. The graph was hosted on an http server, which could recieve http requests from clients. Note: This assignment was pair-programmed (contributions reflect commits from Stelios's account)

Second iteration: Durability

This iteration adds durability to the graph store by storing the graph on-disk. A logging feature was added, providing failure atomicity.

Third iteration: Reliability via Chain-Replication

Adds reliability via RPC communication between server replicas.

Fourth iteration: Partioned for scalability

Graph-store is partitioned across multiple machines via RPCs
